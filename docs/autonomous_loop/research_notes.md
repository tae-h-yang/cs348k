# Research Notes

## MoVer

MoVer is a motion-graphics verification system from Stanford that uses a
first-order-logic DSL to check spatio-temporal properties of generated SVG
animations. Its pipeline turns a prompt into both an animation and a verifier;
failed predicates are fed back for iterative correction. The project page
reports that GPT-4o pass@0 was 58.8 percent and rose to 93.6 percent with up to
50 correction iterations on a 5600-prompt benchmark.

Source: https://mover-dsl.github.io/

Relevance: use the same structure, not the same predicates. For humanoids, the
DSL should check root motion, timing, posture, limb role, expected contacts, and
forbidden contacts.

## MotionBricks

MotionBricks is a SIGGRAPH 2026 real-time motion generation framework with a
latent backbone and smart primitives. The public project page says the released
system includes an interactive G1 demo and synthetic training pipeline, and that
motions are generated by neural networks without foot-locking, blending,
collision detection, or hand-authored transitions.

Source: https://nvlabs.github.io/motionbricks/

Relevance: this is exactly why a curation/verifier layer is interesting. It
should be framed as an interface around a generator, not as replacing the
generator.

## PhysDiff

PhysDiff argues that motion diffusion models often produce artifacts such as
floating, foot sliding, and ground penetration, and uses a physics projection
module during diffusion to improve physical plausibility.

Source: https://arxiv.org/abs/2212.02500

Relevance: our checks should include foot skate, penetration, and support/contact
consistency, not only torque demand.

## SONIC

SONIC is a recent humanoid whole-body motion tracking system. The arXiv summary
describes large-scale motion tracking and practical interfaces for natural
humanoid control.

Source: https://arxiv.org/abs/2511.07820

Relevance: controller rollout is a stronger validation target than inverse
dynamics. In this repo, the SONIC bridge is approximate, so it should be used as
an experimental signal until native validation is available.

## AToM

AToM uses GPT-4Vision rewards to improve event-level text-to-motion alignment.

Source: https://atom-motion.github.io/

Relevance: rendered-video review can catch semantic/event failures that numeric
motion proxies miss. It should augment, not replace, physics checks.

## RoboForge

RoboForge couples text-guided motion generation with physical plausibility
optimization for Unitree G1 and uses optimized rollouts to improve the generator
distribution.

Source: https://arxiv.org/abs/2603.17927

Relevance: a stronger future version of this project should move from screening
to training or preference tuning using controller/physics labels.
