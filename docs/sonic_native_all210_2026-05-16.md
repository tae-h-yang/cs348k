# Native SONIC All-210 Validation, 2026-05-16

## Purpose

Validate every exposed local MotionBricks K=1/K=8 reference through the native
SONIC C++ deployment path plus official MuJoCo simulator qpos logging.

This is the current strongest execution evidence. It replaces earlier
Python-only or debug-stream SONIC checks for final claims.

## Selector Analysis

Across all 105 paired K=1/K=8 identities:

| selector | strict pass | survive | K=8 selected | interpretation |
|---|---:|---:|---:|---|
| K1 baseline | 72/105 | 81/105 | 0/105 | strong baseline |
| K8 ID-screened | 74/105 | 83/105 | 105/105 | small gain, not a complete method |
| best pre-controller score | 71/105 | 83/105 | 85/105 | not better than K8 or K1 on strict pass |
| learned native-strict LOMO | 73/105 | 81/105 | 53/105 | too weak for a model claim |
| lowest inverse-dynamics risk | 70/105 | 82/105 | 81/105 | lower risk does not imply best execution |
| native oracle upper bound | 85/105 | 90/105 | 54/105 | shows headroom for controller-in-loop curation |

Interpretation:

- K=8/inverse-dynamics selection is not enough by itself.
- Native SONIC acceptance provides meaningful headroom: 85/105 strict pass if
  the better K=1/K=8 candidate is chosen by native outcome.
- The realistic method claim is controller-in-the-loop curation, not generator
  fine-tuning and not unconditional K-scaling.

## Claim Boundary

Supported:

- MotionBricks upright references include a large executable subset under native
  SONIC.
- Some MotionBricks categories are brittle even when generated by the same
  system.
- Contact/dynamics/MotionSpec metrics are useful pre-controller signals.
- Native controller-in-the-loop curation is the strongest current route to
  physically reasonable selected motion data.

Not supported:

- True MotionBricks fine-tuning.
- Arbitrary text-prompt-to-robot success.
- Crawling or low-posture success with the current controller/gate.
- A learned selector that replaces native rollout.
- A claim that K=8 or inverse dynamics alone reliably improves execution.
